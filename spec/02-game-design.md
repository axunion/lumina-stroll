# 02. ゲームデザイン詳細

このドキュメントは **挙動** の「正」である。数値はすべて `03-reference.md` の CONFIG・パレット・ワールドデータをキー名で参照する（値をここに書かない）。

## 1. ワールドとカメラ

- ワールドは横長の 1 枚（`CONFIG.worldWidth` × `CONFIG.worldHeight`、world 座標 = CSS px）。
- カメラはプレイヤーを画面中央に置き、ワールド端では画面がワールド外を映さないようにクランプする（画面がワールドより大きい軸は中央寄せ）。
- プレイヤーもワールド端でクランプする（跳ね返りなし、めり込みなし。壁に沿って滑る）。
- 背景は手続き的なタイルグリッド（`CONFIG.tileSize`）。市松模様で `background` 色と `tileAccent` 色（バイオームブレンド後、§8）を交互に塗る。**可視範囲のタイルのみ** 描画する（カメラ位置から開始インデックスを計算）。

## 2. 移動

- グリッドフリー。入力ベクトル（上下左右の合成）を **正規化** し、`CONFIG.playerSpeed × dt秒` を掛けて位置に加算する。斜め移動が速くならない。
- 加速・慣性・イージングは **なし**（即時速度）。
  - 決定の記録: 慣性つき移動も検討したが、非ゲーマーには「思ったところで止まらない」操作感がストレスになるため却下。「柔らかさ」は移動ではなくパーティクルと光の表現で出す。
- プレイヤーの見た目: 半径 `CONFIG.playerRadius` の柔らかく光る円（放射グラデーション）。`player` スプライトがあればその上に重ねて描く（§10 — グロー円は常に描く）。

## 3. 入力

- キーボード: `window` の keydown/keyup で `e.code` を `Set<string>` に出し入れする。対象コード: `ArrowUp` / `ArrowDown` / `ArrowLeft` / `ArrowRight` / `KeyW` / `KeyA` / `KeyS` / `KeyD`。
- モバイル D-pad: 4 ボタン（`04-ui.md` §4）。各ボタンは `pointerdown` で対応フラグを true、`pointerup` / `pointercancel` / `pointerleave` で false にする `{ up, down, left, right }` の plain object を持つ。
- 毎フレーム、キー Set と D-pad フラグを **OR でマージ** して入力ベクトルを作る（どちらで操作しても同じ挙動）。
- D-pad ボタンには `touch-action: none` を指定（スクロール・ダブルタップズームの誤発動防止）。
- `gameState.isMenuOpen` が true の間、入力は無視される（update 自体がスキップされる — `01-architecture.md` §5）。

## 4. ライティング

技法を以下に **固定** する:

1. offscreen の light canvas 全面を `rgba(0, 0, 0, CONFIG.darknessAlpha)` で塗りつぶす。
2. `globalCompositeOperation = 'destination-out'` に切り替え、光源ごとに放射グラデーション（中心: 不透明 → `半径 + CONFIG.lightSoftness` で透明）の円を塗って暗幕に「穴」を開ける。
   - プレイヤー: 半径 `CONFIG.lightRadius`
   - 点灯済みブレイジャー（画面内のみ）: 半径 `CONFIG.brazierLightRadius`
3. composite を戻し、light canvas を main canvas に `drawImage` で重ねる。

- 決定の記録: 単一の放射グラデーションを直接 main canvas に塗る方式は光源が 1 つしか扱えず、`destination-out` の穴あけ方式が **複数光源の重なりを正しく合成できる唯一の単純な技法** であるため採用。
- プレイヤーの光は sine で ±4% ゆっくり脈動する（`reducedMotion` 時は無効、§9）。
- 光の色味: 穴あけの後、light canvas 全体に薄く `lightTint`（バイオームブレンド後）を乗せてもよいが、MVP では省略可（やる場合も彩度は極めて低く）。

## 5. パーティクル

- **単一のプール配列** で全種類を管理する。各要素は `kind` を持つ: `footprint` / `burst` / `flame` / `ambient`（型は `03-reference.md`）。
- 総数が `CONFIG.maxParticles` に達したら新規スポーンを抑制する（最古を消すのではなく、スポーンをスキップ）。
- update: 経過時間で寿命判定（超過したら配列から除去）、速度で位置更新、寿命比率で alpha を減衰。
- render: `globalAlpha` を設定した塗りつぶし円（`ctx.arc`）。放射グラデーションは使わない（パーティクル数 × グラデーション生成は高コスト。柔らかさは低 alpha の重なりで出す — 決定として固定）。

| kind | スポーン条件 | 挙動 |
|---|---|---|
| `footprint` | プレイヤー移動中、`CONFIG.particleSpawnInterval` ms ごと | プレイヤー足元に生成、ゆっくり上に漂いフェードアウト |
| `burst` | クリスタル取得の瞬間に `CONFIG.burstParticleCount` 個 | 取得位置から放射状にランダム方向へ飛散、減速しつつフェード |
| `flame` | 点灯済みブレイジャーごとに連続的に少量 | 炎の根元から上昇、コア色 → 先端色へ変化しつつ揺らめく |
| `ambient` | 起動時に `CONFIG.ambientParticleCount` 個を常駐 | 画面内をゆっくり漂う。寿命が尽きたら可視範囲内で再生成。色はバイオームブレンド色（§8） |

## 6. マジカルクリスタル

- 未取得のクリスタルは sine で上下にボブする。個体ごとに `phase` オフセットを持ち、同期して動かない。
- 取得判定: 毎フレーム、未取得クリスタルとプレイヤーの距離平方を比較（`distSq < CONFIG.crystalPickupRadius²`。平方根を取らない）。
- 取得時（1 回だけ発生する離散イベント）:
  1. `collected = true` にする（以後レンダリングも判定もしない。リスポーンなし）
  2. `burst` パーティクルをスポーン
  3. フローティングテキスト "+1" を生成（`CONFIG.floatingTextDurationMs` かけて上昇 + フェードアウト）
  4. `collectCrystal()` を呼ぶ

## 7. ブレイジャー

- 未点灯: 小さく暗い残り火のドット（かすかに視認できる程度）。
- プレイヤーが `CONFIG.brazierProximity` 内に入ったら **自動点灯**（ボタン不要）。1 回だけの離散イベント:
  1. `lit = true`、`litAt = now` を記録
  2. `lightBrazier()` を呼ぶ
- 点灯後:
  - 青紫の炎（`flame` パーティクル + コアのグロー円）を描画し続ける
  - `destination-out` の恒久光源になる（§4）。点灯直後は `litAt` から約 600ms かけて光半径を 0 → `CONFIG.brazierLightRadius` にイーズイン（ease-out カーブ）
  - **決して消えない**（セッション内永続 — `00-overview.md` §4）

## 8. バイオーム

- バイオームは x 範囲で決まる（定義は `03-reference.md`）: Enchanted Forest → Crystal Cave。
- **描画色は連続ブレンド**: 毎フレーム、プレイヤー x からブレンド係数を計算する。

```
t = smoothstep01((player.x - (CONFIG.biomeBoundaryX - CONFIG.biomeBlendWidth / 2)) / CONFIG.biomeBlendWidth)
```

  `t = 0` で Forest、`t = 1` で Cave。`background` / `tileAccent` / `ambientParticle` / `lightTint` の全チャンネルを `lerpColor` で補間する。ハードスワップ禁止。
- **`currentBiome` Signal は離散切替**: `biomeIdAt(player.x)` は `CONFIG.biomeBoundaryX` を境に ID を返し、前回値と変わった時だけ `setCurrentBiome()` を呼ぶ（HUD のバイオーム名表示用）。
- 描画の連続 lerp と Signal の離散切替は **意図的に分離** されている。HUD の名前は境界で 1 回だけ切り替わり、画面の色は境界帯（`biomeBlendWidth`）の間ずっと滑らかに変わる。これを「不整合」として同期させてはならない。

## 9. reduced-motion 挙動マトリクス（正規表）

`gameState.reducedMotion` が true の時の各エフェクトの挙動。この表が正であり、実装・QA はこの表に従う。

| エフェクト | 通常 | reduced-motion 時 |
|---|---|---|
| footprint パーティクル | `particleSpawnInterval` ごとにスポーン | **無効**（スポーンしない） |
| クリスタル取得バースト | `burstParticleCount` 個のパーティクル | パーティクルなし。**広がるリングのストローク 1 本** で代替（取得フィードバックは残す） |
| ambient パーティクル | 常駐・漂う | **無効** |
| クリスタルのボブ | sine で上下 | **静止**（phase 位置で固定） |
| ブレイジャーの炎 | フリッカー + `flame` パーティクル | **定常グロー**（揺らがない光の円のみ、パーティクルなし） |
| "+1" フローティングテキスト | 上昇しながらフェード | **その場でフェード**（移動しない） |
| プレイヤー光のパルス | sine ±4% | **無効**（固定半径） |
| バイオーム色 lerp | 有効 | **有効のまま**（位置に依存する状態表現でありアニメーションではない） |
| Kobalte Dialog の出入りアニメーション | CSS transition | **無効**（`04-ui.md` §6） |
| ウィスプ | 漂い + プレイヤーへのゆるい追従 | **無効**（描画・更新ともに停止。ambient パーティクルと同じ扱い） |
| 光る草の近接グロー | 距離に応じて明るさ変化 | **有効のまま**（バイオーム色 lerp と同じく、位置に依存する状態表現でありアニメーションではない） |
| 碑文テキスト | 上昇しながらフェード | **その場でフェード**（"+1" テキストと同規則） |

- 初期値は `prefers-reduced-motion: reduce` の自動検出。設定メニューのトグルでいつでも変更可能（`01-architecture.md` §3.3）。
- トグルは即時反映される（次のフレームからマトリクスに従う）。既存のパーティクルは自然消滅に任せてよい（即時消去は不要）。
- スプライトの有無はこの表に影響しない。ボブの静止・フリッカーの停止などは座標・半径の計算に適用されるため、スプライト描画でも手続き描画でも同じように効く。

## 10. スプライト描画とフォールバック

見た目の差し替えの仕組み。マニフェスト（キー・パス・サイズ・アンカー）は `03-reference.md` §7 が正。

- **ディスパッチ規則**: 各 render 関数は描画時に `SpriteMap` を参照する。対応キーの画像がロード済みならスプライトを `drawImage` で描き、なければ **従来の手続き描画をそのまま** 行う。手続き描画がフォールバックの「正」であり、スプライト導入によって変更しない。
- **受け入れ基準（フラッグシップの体験則）**: `public/assets/player.png` を置いてリロードすると使われる。ファイルを消してリロードすると手続き描画に戻る。以上。ビルド・設定変更・コード変更は一切不要。
- **座標と変換**: スプライトは既存のワールド座標 + 既存の変換をそのまま使う。クリスタルのボブオフセットはスプライトの y に適用される。アンカー（`SpriteDef.anchor`）で描画原点を決める（計算は `assets.ts` の `spriteDrawOrigin` — `03-reference.md` §6.3）。
- **ハロー維持則**: player / crystal / 点灯ブレイジャーは、スプライトの **背後に既存の手続きグローを常に描く**。
  - 決定の記録: `CONFIG.darknessAlpha` の暗幕下では、グローのないスプライトは平板に沈んで見える。世界観の「発光」はハローが担い、スプライトは形状のみを差し替える。素材側にグローを焼き込むことは禁止する（`08-asset-guide.md` §2 — 二重発光防止）。
  - 未点灯ブレイジャーは例外: スプライトをそのまま描く（追加グローなし。暗い見た目は素材側が担う）。
  - ブレイジャーのフリッカーはグロー半径にのみ適用し、スプライト自体は拡縮しない（reduced-motion の「定常グロー」とも自然に整合する）。
- **タイルは all-or-nothing**: `tileForest` と `tileCave` の **両方** が揃った時のみテクスチャモードになる。どちらか一方でも欠けたら市松フォールバック。
  - テクスチャモードでは可視タイルごとに `drawImage`（市松の交互塗り分けはテクスチャ自体の変化に置き換わる）。
  - バイオームブレンド帯では、タイルごとに forest タイルを描いた上に cave タイルを `globalAlpha = t`（そのタイル位置の `biomeBlendAt`。色 lerp と同じ係数）で重ね描きする。
  - 決定の記録: 片方だけのテクスチャでは境界帯の中間表現が定義できないため、all-or-nothing とする。
- **常に手続きのまま（スプライト対象外）**: パーティクル全種・取得リング・フローティングテキスト・ライティング・ウィスプ。
  - 決定の記録: `CONFIG.maxParticles` 上限のドット群への `drawImage` は `ctx.arc` より高コスト。また光の粒・光の生き物は手続きグローの方が世界観に整合する。
- **ライティング不変**: 暗幕 + `destination-out` の穴あけ（§4）はスプライトの有無と完全に独立であり、変更しない。合成順も従来どおり最上段。
- **ロード挙動**: 起動時に全キーのロードを開始し、成功した画像から順次描画に使われる（プログレッシブ差し替え。ローディング画面なし）。404・decode 失敗は握りつぶしてフォールバック続行（`01-architecture.md` §8）。
- **アニメーションフレームなし**: スプライトは 1 枚静止画のみ。生命感はボブ・フリッカー・点灯イーズという既存の変換が与える（`00-overview.md` §7）。

## 11. 進捗セーブ

体験の柱 3「失われるものは何もない」のリロード越え拡張。スキーマと検証規則は `03-reference.md` §8 が正。

- **保存対象**: 取得クリスタル id・点灯ブレイジャー id・発見碑文 id・`audioMuted`。これ以外（プレイヤー位置・reduced-motion 等）は保存しない（`00-overview.md` §4）。
- **書き込みタイミング**: 離散イベント（取得・点灯・発見・ミュート切替）の瞬間のみ、同期的に write-through する。フレーム毎の書き込みは禁止。
- **起動時復元**: モジュール初期化で load + 検証し、store の初期値とワールド配列（collected / lit / discovered フラグ）に反映する。復元されたブレイジャーは **点灯イーズを再生せず即時フル半径** で点灯している（`litAt` はイーズ完了済みとして扱う）。
- **壊れたセーブ**: JSON 破損・未知バージョン・型不一致は **黙って新規開始**（null 扱い）。クラッシュ・エラー UI は出さない。localStorage 自体が使えない環境（Safari プライベートモード等）でも同様に新規開始で動き続ける。
- **リセット**: Settings タブの 2 段階確認ボタン（`04-ui.md` §3.2）→ セーブを削除して `location.reload()`。
  - 決定の記録: canvas ローカルのワールド配列をイベント配線なしで確実に初期状態へ戻す最小手段としてページリロードを採用。

## 12. サウンド

Web Audio による生成音のみ。パラメータ（周波数・ゲイン・時間）は `03-reference.md` §9 が正。

- **アンビエントベッド**: バイオームごとに 1 系統。
  - Enchanted Forest: 暖かい低音ドローン + フィルターを通したノイズの葉ずれ。
  - Crystal Cave: より深いドローン + まれに響く正弦波の雫。
- **クロスフェード**: 2 つのベッドを equal-power でクロスフェードする。係数は **バイオーム色 lerp と同一の t**（§8 の `biomeBlendAt`）。ゲイン変更はランプ（`setTargetAtTime`）で行い、ステップ変化させない。
- **ワンショット**: クリスタル取得チャイム / ブレイジャー点灯スウェル / 碑文発見ベルの 3 種。離散イベント時に鳴らす。
- **開始**: AudioContext は **最初のユーザージェスチャー**（keydown / pointerdown の once リスナー）で生成・resume する。それまでは完全に無音（ブラウザの autoplay 制限に従う）。
- **ミュート**: Settings のトグル（`04-ui.md` §3.2）。マスターゲインをランプで 0 にする（ノードは止めない）。値は `audioMuted` としてセーブされる（§11）。
- **reduced-motion と独立**: 音は「動き」ではない。`reducedMotion` はサウンドに影響しない（明示的な決定）。
- **メニュー中も継続**: アンビエントは `isMenuOpen` 中も鳴り続ける。update がスキップされるため、ワンショットは自然に発生しない（特別扱い不要）。
- 決定の記録（生成音 vs 素材ファイル）: 画像と揃えて音声ファイル差し替えも検討したが、ループ素材の制作・検品コストが画像より高く、生成ドローンで「静かで癒される」トーンには十分なため生成音を採用。音声ファイル対応への布石は仕込まない（`00-overview.md` §7）。

## 13. 小さな発見

歩くだけで見つかる演出群。「操作を覚えなくても歩けば何かが起こる」（柱 2）に従い、すべて近接のみで発動する。数値は `03-reference.md` §2、配置は同 §4 が正。

### 13.1 ウィスプ

- `CONFIG.wispCount` 体がワールドに常駐し、`CONFIG.wispWanderSpeed` でゆっくり漂う（進行方向は滑らかに変わり続ける）。
- プレイヤーが `CONFIG.wispNoticeRadius` 内に入ると気づき、`CONFIG.wispFollowSpeed` でゆるやかに引き寄せられて付かず離れず漂う。離れると徘徊に戻る。急加速・急旋回はしない（柱 1）。
- 純粋な視覚要素: state なし・セーブなし・HUD なし。canvas ローカルで完結する。描画は手続きグロー（§10 — スプライト対象外）。色はバイオームブレンド後の `ambientParticle` 色。

### 13.2 光る草

- `PLANTS`（`03-reference.md` §4.4）に固定配置。バイオームは各個体の x 座標で決まる（スプライトキー `plantForest` / `plantCave` の選択に使う）。
- 明るさはプレイヤー距離の関数: `proximityGlow01(distSq, CONFIG.plantGlowInnerRadius, CONFIG.plantGlowOuterRadius)`（`03-reference.md` §6.1）。近づくほど明るく、離れても **微光が残る**（完全消灯しない — 下限は `CONFIG.plantGlowFloor`）。変化は関数値そのもので滑らか（追加のイージング不要）。
- 取得・カウント・セーブはない。何度でも明るくなる、ただの優しい応答。

### 13.3 碑文

- `INSCRIPTIONS`（`03-reference.md` §4.5）に固定配置された石碑。未発見でもかすかに視認できる。
- プレイヤーが `CONFIG.inscriptionProximity` 内に入ると、石碑の上に詩文がフローティングテキストで表示される（`CONFIG.inscriptionTextDurationMs`。§9 の "+1" と同じ上昇 + フェード規則）。表示は **エッジトリガー**: 近接圏に入った瞬間に 1 回表示し、圏外に出てから再度入れば再表示される（再訪のたび読める）。
- **初回のみ** 離散イベント「発見」が発生する: `discoverInscription(id)` を呼び、セーブに書き込む（§11）。二度目以降の再表示はイベントを発生させない。
- Journal タブに発見済み碑文の詩文が並ぶ。未発見の碑文はロック行（"…"）として総数だけ見える（`04-ui.md` §3.1）。
