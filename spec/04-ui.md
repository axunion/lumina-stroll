# 04. UI 仕様 — HUD / Kobalte / CSS Modules

このドキュメントは UI 構造・CSS クラス名・アクセシビリティ実装の「正」である。色の RGB 値は `03-reference.md` が正（本書はトークン名へのマッピングのみ定義する）。

## 1. レイヤースタック（z-index を固定）

| レイヤー | 所有ファイル | z-index |
|---|---|---|
| canvas（`canvasWrapper` 内） | GameCanvas.tsx | `0` |
| HUD（上部） | GameUI.tsx | `10` |
| モバイル D-pad | GameCanvas.tsx（`01-architecture.md` §4） | `10` |
| Kobalte Dialog（Portal → `<body>`） | GameUI.tsx | `50`（overlay / content とも） |

- `App.tsx` のルート要素は `class={styles.appRoot}`（`position: relative`、全画面）。
- Dialog は `<body>` に Portal されるため `appRoot` のスタッキングコンテキスト外にある。overlay / content に `z-index: 50` を直接指定する。

## 2. HUD

左上に横並びのピル（丸角チップ）列、右上に設定ボタン。

- **クリスタルピル**: `Sparkles` アイコン + `{gameState.crystalsCollected} / {TOTAL_CRYSTALS}`
- **ブレイジャーピル**: `Flame` アイコン + `{gameState.litBraziersCount} / {TOTAL_BRAZIERS}`
- **バイオームチップ**: `gameState.currentBiome` に対応する `Biome.name`（`BIOMES` から引く）
- **設定ボタン**（右上）: `Settings` アイコンのみのボタン。`onClick={() => setMenuOpen(true)}`。

アイコンの規則:

- 個別パス import: `import Sparkles from 'lucide-solid/icons/sparkles'`（`01-architecture.md` §7）
- `size={20}`、`aria-hidden="true"` を付け、隣に `class={styles.srOnly}` の視覚非表示テキストラベルを置く（例: "Crystals collected"）。アイコンのみのボタンは `aria-label` を付ける。

## 3. Dialog（設定 / コレクションログ）

単一の Kobalte Dialog を制御コンポーネントとして使う。`Dialog.Trigger` は使わない（トリガーは HUD の plain button — open が store 制御のため）。

```tsx
<Dialog.Root open={gameState.isMenuOpen} onOpenChange={setMenuOpen}>
  <Dialog.Portal>
    <Dialog.Overlay class={styles.dialogOverlay} />
    <div class={styles.dialogPositioner}>
      <Dialog.Content class={styles.dialogContent}>
        <div class={styles.dialogHeader}>
          <Dialog.Title class={styles.dialogTitle}>Menu</Dialog.Title>
          <Dialog.CloseButton class={styles.dialogClose} aria-label="Close menu">
            <X size={20} aria-hidden="true" />
          </Dialog.CloseButton>
        </div>
        <Tabs.Root defaultValue="journal">
          <Tabs.List class={styles.tabsList}>
            <Tabs.Trigger class={styles.tabsTrigger} value="journal">Journal</Tabs.Trigger>
            <Tabs.Trigger class={styles.tabsTrigger} value="settings">Settings</Tabs.Trigger>
            <Tabs.Indicator class={styles.tabsIndicator} />
          </Tabs.List>
          <Tabs.Content class={styles.tabsContent} value="journal">…</Tabs.Content>
          <Tabs.Content class={styles.tabsContent} value="settings">…</Tabs.Content>
        </Tabs.Root>
      </Dialog.Content>
    </div>
  </Dialog.Portal>
</Dialog.Root>
```

### 3.1 Journal タブ（コレクションログ）

- クリスタル: `{crystalsCollected} / {TOTAL_CRYSTALS}` の集計行（`logList` / `logItem`）。個別クリスタルの発見リストは作らない（store は集計数しか持たない — 契約の 7 フィールドを守る）。
- ブレイジャー: 同様に集計行。
- バイオーム: `BIOMES` の一覧を表示し、`currentBiome` に到達済みマークを付ける（`logItemFound`）。
  - 注: 「訪れたことがあるか」の履歴は持たない（state 追加禁止）。表示は「現在いるバイオーム」の強調のみとする。
- 碑文: 集計行 `{discoveredInscriptionIds.length} / {TOTAL_INSCRIPTIONS}` に続けて、`INSCRIPTIONS`（`03-reference.md` §4.5）を id 順に一覧表示する。
  - 発見済み（`discoveredInscriptionIds` に id が含まれる）: 詩文本文を `inscriptionText` で表示。
  - 未発見: ロック行（`logItemLocked`）。内容は "…" のみ、`aria-label="Undiscovered inscription"`。どこにあるかのヒントは出さない（歩いて見つけるもの）。

### 3.2 Settings タブ

- **Reduce motion** トグル: Kobalte `Switch`。`checked={gameState.reducedMotion}` / `onChange={setReducedMotion}`。

```tsx
<Switch.Root class={styles.switchRoot} checked={gameState.reducedMotion} onChange={setReducedMotion}>
  <Switch.Label class={styles.switchLabel}>Reduce motion</Switch.Label>
  <Switch.Input class={styles.srOnly} />
  <Switch.Control class={styles.switchControl}>
    <Switch.Thumb class={styles.switchThumb} />
  </Switch.Control>
</Switch.Root>
```

- **Sound** トグル: Kobalte `Switch`（上と同じ `switchRoot` 系クラスを再利用）。`checked={!gameState.audioMuted}` / `onChange={(on) => …}`（ハンドラー内で `setAudioMuted(!on)` + `persistence.setAudioMuted(!on)` + `audio.setMuted(!on)` — `01-architecture.md` §3.2 のイベント 5）。ラベルは "Sound"。
- **遊び方テキスト**: 静的な短文（"Walk with arrow keys or WASD. Touch crystals to collect them. Warm braziers light as you pass."）。
- **リセットボタン**（タブ最下部）: 2 段階確認の plain button。Kobalte AlertDialog は導入しない（新規依存面を増やさない決定）。
  - 通常状態（`resetButton`）: ラベル "Begin a new stroll"。押下で armed 状態に入る。
  - armed 状態（`resetButton` + `resetButtonArmed`）: ラベル "Press again — your journey will start over"。`RESET_ARMED_DURATION_MS`（`03-reference.md` §2.1）経過で通常状態に自動復帰。armed 中の再押下で実行: `persistence.clear()` → `location.reload()`（`01-architecture.md` §10）。
  - armed 状態はコンポーネントローカルの Signal（store に入れない — UI の一時状態であり契約の 7 フィールド外）。
  - ラベルの切り替えは `aria-live="polite"` の領域で通知する（§6）。
  - 文言は「失う」ではなく「新しく歩き直す」トーンを保つ（`00-overview.md` §2 柱 3 への配慮 — リセットは唯一進捗が消える操作なので、二段階 + 明示的文言で誤操作を防ぐ）。

## 4. モバイル D-pad

- GameCanvas.tsx の wrapper 内、右下に配置。3×3 グリッドの十字位置に 4 ボタン（上下左右）。
- アイコン: `ChevronUp` / `ChevronDown` / `ChevronLeft` / `ChevronRight`（個別 import、`aria-hidden`）。各ボタンに `aria-label`（"Move up" 等）。
- 半透明（背景 alpha ~0.25、押下中は濃く）。`touch-action: none`。
- 表示条件: `@media (pointer: coarse)` でのみ表示（デスクトップでは非表示）。

## 5. CSS Modules クラス一覧（`Game.module.css`）

クラス名は **最初から camelCase で書く**（`camelCaseOnly` 変換は安全網であり、ケバブケースを書かない）。

| クラス | 用途 |
|---|---|
| `appRoot` | 全画面ルート。テーマトークン（§5.1）を宣言 |
| `canvasWrapper` | canvas + D-pad のコンテナ（`position: relative`） |
| `gameCanvas` | `display: block; width: 100%; height: 100%` |
| `hud` | HUD 行のレイアウト（上部、`pointer-events: none`、子ボタンのみ `auto`） |
| `hudItem` | ピル 1 個（半透明ダーク背景、丸角） |
| `hudIcon` / `hudValue` | ピル内のアイコン / 数値 |
| `biomeChip` | バイオーム名チップ |
| `settingsButton` | 右上のアイコンボタン |
| `dpad` / `dpadButton` | D-pad グリッド / 各ボタン |
| `dialogOverlay` | 全画面の半透明暗幕（z-50） |
| `dialogPositioner` | `position: fixed; inset: 0`、中央寄せ flex（z-50） |
| `dialogContent` | ダイアログ本体（丸角、ダーク、最大幅 ~28rem） |
| `dialogHeader` / `dialogTitle` / `dialogClose` | ヘッダー行 / タイトル / 閉じるボタン |
| `tabsList` / `tabsTrigger` / `tabsIndicator` / `tabsContent` | Kobalte Tabs 各パーツ |
| `logList` / `logItem` / `logItemFound` | Journal の一覧 / 行 / 到達済み強調 |
| `logItemLocked` | 未発見碑文のロック行（減光表示） |
| `inscriptionText` | 発見済み碑文の詩文（イタリック等、静かな装飾） |
| `resetButton` / `resetButtonArmed` | リセットボタンの通常 / armed 状態 |
| `switchRoot` / `switchLabel` / `switchControl` / `switchThumb` | Kobalte Switch 各パーツ |
| `srOnly` | 視覚非表示・スクリーンリーダー可読（定番の clip パターン） |

### 5.1 テーマトークン

`:root`（`index.css`）に CSS カスタムプロパティとして宣言する。値は `03-reference.md` のパレットに対応させる（UI は Cave 寄りの落ち着いた色をベースにする）。

- 注: Kobalte の `Dialog.Portal` は `<body>` 直下にダイアログをレンダリングし、`appRoot` の DOM 子孫にならない。CSS カスタムプロパティは DOM の祖先関係に沿って継承されるため、`.appRoot` に宣言すると Portal 先のダイアログには継承されず、文字色・背景色が未定義になる。`:root` に宣言すればどちらの subtree からも参照できる。

| トークン | 対応 |
|---|---|
| `--colorBgDeep` | Cave `background` を基にした最暗色 |
| `--colorSurface` | HUD ピル / ダイアログ背景（半透明ダーク） |
| `--colorText` | 明るいオフホワイト（"+1" テキスト色と同系） |
| `--colorAccent` | クリスタル色 `[140, 235, 255]` 系 |
| `--colorFlame` | 炎コア色系（Flame アイコン等に使用） |

## 6. アクセシビリティ

- **Kobalte の既定挙動を壊さない**: フォーカストラップ、`aria-modal`、Escape で閉じる、閉じた後のフォーカス復帰はすべて Kobalte に任せる。上書き・自前実装禁止。
- Dialog overlay / content の出入りアニメーション（フェード等）は次の **両方** で無効化する:
  - `@media (prefers-reduced-motion: reduce)`（OS 設定）
  - `.appRoot`（および Portal 先で参照できるよう `dialogOverlay` / `dialogContent` 自体）に付与する `data-reduced` 属性（store トグル由来）。GameUI が `gameState.reducedMotion` を属性にバインドする。
- canvas 要素: `role="img"` + `aria-label="A glowing dungeon world. Use arrow keys or WASD to stroll."`
- すべてのアイコンボタンに `aria-label`。装飾アイコンは `aria-hidden`。
- フォーカスリングを消さない（`:focus-visible` で見えるリングを維持）。
- リセットボタンの armed 状態のラベル変化は `aria-live="polite"` で読み上げられるようにする（§3.2）。
- サウンドの開始に専用 UI は不要 — 最初のキー入力かタップで自動的に始まる（`01-architecture.md` §9）。ミュートトグルが唯一のサウンド UI。
