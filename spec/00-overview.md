# 00. 概要 — Lumina Stroll

## 1. コンセプト

**Lumina Stroll** は、ゲームをほとんど遊ばない人のための、癒し系 2D ダンジョン探索ウォーキングシムである。プレイヤーは淡く光る地下世界をただ歩き回るだけで、美しくインタラクティブな体験を得られる。複雑なルールも、失敗も、プレッシャーもない。トーンは「静かで、癒される」。

描画は HTML5 Canvas 上の手続き的レンダリングを基盤とする。`public/assets/` に画像素材（PNG）が置かれているエンティティはスプライトとして上書き描画され、素材がなければ自動的に手続き描画へフォールバックする（`02-game-design.md` §10）。手続き描画は常に完全な表現として維持され、素材はオプションの上書きレイヤーである。音は Web Audio による生成音のみで、音声素材ファイルは使わない。

## 2. 体験の柱（判断基準）

実装中に迷ったら、この 3 つを優先順に適用して判断する。

1. **迷ったら、より遅く・より柔らかく** — 急な変化・点滅・硬い動きより、ゆっくりしたフェードと滑らかな補間を選ぶ。
2. **操作を覚えなくても歩けば何かが起こる** — すべてのギミック（クリスタル・ブレイジャー・バイオーム変化）は「近づく／触れる」だけで発動する。ボタン操作を要求しない。
3. **失われるものは何もない** — 一度得た進捗（取得したクリスタル、点灯したブレイジャー）は決して失われない。減るカウンター・消える光は存在しない。

## 3. 存在しないもの（明示的な非要件）

以下はこのゲームに **存在しない**。実装してはならない。

- 戦闘・敵・ダメージ
- タイマー・制限時間
- ゲームオーバー・失敗状態
- スコアによるプレッシャー（ランキング・評価・ペナルティ）
- 落下・障害物などプレイヤーを妨げる要素

## 4. スコープ（機能一覧）

- グリッドフリーの滑らかな移動（矢印キー + WASD + 画面上のモバイル D-pad）
- 動的ライティング（暗幕 + プレイヤー周囲の柔らかい放射状の視界）
- パーティクル（足跡・クリスタル取得バースト・ブレイジャーの炎・環境の漂う光）
- マジカルクリスタル: 接触で取得 → バースト + "+1" フローティングテキスト + カウント加算
- ブレイジャー: 近接で自動点灯（青紫の炎）→ その一帯を恒久的に照らす
- バイオーム: プレイヤーの x 座標に応じて背景色と環境パーティクル色が滑らかに補間される（Enchanted Forest → Crystal Cave）
- HUD（取得数・点灯数・現在バイオーム）と設定メニュー（Kobalte Dialog + Tabs）
- `prefers-reduced-motion` の自動検出 + 設定トグル
- スプライト差し替え: `public/assets/` に置いた PNG でプレイヤー・クリスタル・ブレイジャー・タイル・光る草・碑文の見た目を差し替え（素材がなければ手続き描画に自動フォールバック — `02-game-design.md` §10）
- 進捗セーブ: 取得クリスタル・点灯ブレイジャー・発見碑文・サウンド設定を localStorage に保存し、リロード後も保持（`02-game-design.md` §11）
- サウンド: Web Audio 生成のバイオーム連動アンビエント + 取得・点灯・発見のワンショット音、ミュートトグル（`02-game-design.md` §12）
- 小さな発見: ウィスプ（漂う光の生き物）・光る草（近づくと明るくなる）・碑文（見つけると Journal に記録される短い詩文）（`02-game-design.md` §13）

### 永続化の解釈（固定）

進捗（取得クリスタル・点灯ブレイジャー・発見碑文）とサウンド設定は localStorage によりリロードをまたいで保持される（`02-game-design.md` §11）。一度点灯したブレイジャーは二度と消えない。**プレイヤー位置は保存しない** — 散歩は毎回入口から始まる（意図的な決定: 進捗だけが積み重なり、道のりは毎回歩き直す）。

## 5. 用語集

| 用語 | 意味 |
|---|---|
| world 座標 | ワールド空間の座標。単位は CSS px（devicePixelRatio 適用前）。 |
| biome（バイオーム） | ワールドの x 範囲ごとの環境テーマ。色パレットを持つ。 |
| brazier（ブレイジャー） | 篝火台。近接で自動点灯し、恒久光源になる。 |
| crystal（クリスタル） | 接触で取得できる収集アイテム。 |
| footprint particle | プレイヤーの移動軌跡に残る、光って消える足跡パーティクル。 |
| ambient particle | バイオーム色に染まった、漂う環境の光の粒。 |
| burst particle | クリスタル取得時に放射状に飛び散るパーティクル。 |
| lit（点灯） | ブレイジャーが点火済みの状態。二度と unlit に戻らない。 |
| 離散イベント | フレーム毎ではなく「起きた瞬間」に 1 回だけ発生する事象（取得・点灯・バイオーム境界越え・碑文発見・ミュート切替）。Signal 更新とセーブ書き込みはこの時のみ行う。 |
| sprite（スプライト） | `public/assets/` の PNG 画像による見た目の上書き。素材がなければ手続き描画のまま。 |
| fallback（フォールバック） | スプライト素材がない（またはロードに失敗した）ときに使われる手続き描画。常に完全な表現として維持される。 |
| wisp（ウィスプ） | ワールドを漂う小さな光の生き物。プレイヤーに気づくとゆるやかに寄ってくる。純粋な視覚要素。 |
| glow plant（光る草） | 固定配置の発光植物。プレイヤーが近づくほど明るくなり、離れても微光が残る。 |
| inscription（碑文） | 石碑に刻まれた短い詩文。近づくと表示され、初回のみ「発見」として Journal に記録される。 |
| セーブスナップショット | localStorage に保存される進捗の全体像（`SaveDataV1`）。`persistence.ts` が所有し、離散イベントごとに書き込む。 |

| `04-ui.md` | HUD / Dialog / Tabs / CSS クラス / アクセシビリティ | UI 構造とクラス名 |

**衝突時の規則**: 数値・型・座標が食い違ったら `03-reference.md` が勝つ。挙動の記述が食い違ったら `02-game-design.md` が勝つ。数値は 03 にのみ記載し、他ドキュメントは `CONFIG.playerSpeed` のようにキー名で参照する（値の重複記載禁止）。

**例外（明示的に 1 つだけ）**: `08-asset-guide.md` のアセット別ブリーフは、AI 生成プロンプトとして単体でコピペできるようにするため、`03-reference.md` §7 のサイズとパレット値を **転記** する。08 の数値は 03 の写しであり、食い違ったら 03 が勝つ。

## 7. スコープ外・将来アイデア（非拘束）

以下は実装しない。実装時にこれらへの布石（抽象化・オプション引数）を仕込むことも禁止する（YAGNI）。

- 第 3 のバイオーム
- ゲームパッド対応
- スプライトのアニメーションフレーム（複数フレーム切替・スプライトシート/アトラス）
- 音声素材ファイルの読み込み（音は Web Audio 生成のみ）
- プレイヤー位置のセーブ（§4「永続化の解釈」）
